# 前端 PWA 设计文档

## 1. 概述

React PWA 移动端应用，作为学生时间规划 Agent 的薄客户端。零业务逻辑，只负责 UI 渲染、用户交互、推送接收。纯手机端体验，不考虑桌面浏览器。

## 2. 技术栈

| 技术 | 说明 |
|------|------|
| React 18+ | UI 框架 |
| Vite | 构建工具 |
| React Router v6 | 路由 |
| Zustand | 状态管理 |
| UI 库 | 实现阶段决定（Ant Design Mobile / Vant React / Tailwind） |
| TypeScript | 类型安全 |

## 3. 页面结构与导航

### 3.1 Tab 导航

底部 tab bar，三个 tab：

```
┌─────────────────────────────┐
│         顶部导航栏            │
├─────────────────────────────┤
│                              │
│         页面内容区域          │
│                              │
├─────────────────────────────┤
│  💬聊天    📅日历    👤我的   │
└─────────────────────────────┘
```

- 聊天页为默认首页
- tab 切换保持各自状态（切到日历再切回聊天，对话不丢失）
- 登录页独立于 tab 体系

### 3.2 顶部导航栏

按页面区分内容，标题居中：

| 页面 | 标题 | 右侧按钮 |
|------|------|----------|
| 聊天页 | Assistant | 无 |
| 日历页 | `3月30日 周一`（当前日期） | `+` 添加任务 |
| 我的页 | 我的 | 无 |

### 3.3 路由结构

```
/login          → 登录页（独立）
/register       → 注册页（独立）
/chat           → 聊天页（默认 tab）
/calendar       → 日历页
/me             → 我的页
/me/courses     → 课表管理
/me/preferences → 偏好设置
/me/notifications → 通知设置
```

## 4. 认证

- JWT token 存 localStorage
- 每次 API 请求带 `Authorization: Bearer <token>`
- WebSocket 连接后第一条消息发 `{ "token": "jwt_token" }`
- token 过期 → 清除 token → 跳转登录页
- 登录页：用户名 + 密码，底部"注册"链接

### 5.2 附件按钮

点击 `📎` 弹出选项：
- 拍照（调用相机）
- 从相册选择
- 选择文件（Excel）

选择后上传到 `POST /api/schedule/upload`，返回解析结果后由 Agent 引导用户确认。

### 5.3 AI 回复 — 进度卡片

Agent 处理过程中，展示动态进度卡片：

```
┌─────────────────────────────┐
│  🤖 正在处理...               │
│                              │
│  ✅ 查询空闲时间              │
│  ✅ 生成复习计划              │
│  ⏳ 等待确认...               │
└─────────────────────────────┘
```

**渲染逻辑：** 前端不预设步骤，完全由 WebSocket 事件驱动：

- 收到 `tool_call` 事件 → 追加一行 `⏳ {工具中文名}...`
- 收到 `tool_result` 事件 → 对应行变为 `✅ {工具中文名}`
- 收到 `done` 事件 → 卡片收起，展示 AI 最终文字回复

**工具名映射表：** 前端维护一个 `tool_name → 中文显示名` 的映射：

| tool_name | 显示名 |
|-----------|--------|
| get_free_slots | 查询空闲时间 |
| create_study_plan | 生成复习计划 |
| parse_schedule | 解析课表 |
| parse_schedule_image | 识别课表图片 |
| list_courses | 查看课表 |
| list_tasks | 查看任务 |
| set_reminder | 设置提醒 |
| recall_memory | 检索记忆 |
| ask_user | 等待确认 |
| 未映射的 tool | 处理中 |

### 5.4 ask_user 结构化卡片

Agent 调用 `ask_user` 时，前端渲染独立卡片，视觉上与普通聊天气泡明确区分：

**confirm 类型（是/否确认）：**
```
┌─────────────────────────────┐
│  确认：4月5日有以下考试？      │
│                              │
│  • 高等数学                   │
│  • 线性代数                   │
│  • 概率论                     │
│                              │
│  [ 确认 ]     [ 取消 ]        │
└─────────────────────────────┘
```

**select 类型（选择题）：**
```
┌─────────────────────────────┐
│  17:00-18:00 已有线代复习。    │
│  你想：                       │
│                              │
│  ○ 把高数改到 18:00-19:00     │
│  ○ 把线代和高数互换时间       │
│  ○ 取消这次修改               │
│                              │
│         [ 确认选择 ]          │
└─────────────────────────────┘
```

**review 类型（展示计划请求确认/修改）：**
```
┌─────────────────────────────┐
│  复习计划如下：                │
│                              │
│  3.30 10:00-12:00 高数极限    │
│  3.31 14:00-16:00 线代矩阵   │
│  4.01 10:00-12:00 概率论基础  │
│  ...                         │
│                              │
│  [ 确认 ]  [ 我想调整 ]       │
└─────────────────────────────┘
```

用户操作后，前端发送 `{ "answer": "..." }` 到 WebSocket。卡片变为不可交互状态（按钮灰掉，显示用户的选择）。

### 5.5 WebSocket 协议

连接：`ws://<host>/ws/chat`

```
→ 客户端发送: { "token": "jwt_token" }
← 服务端返回: { "type": "connected", "session_id": "uuid" }

→ 客户端发送: { "message": "4月5号有三门考试" }
← 服务端推送: { "type": "tool_call", "tool": "get_free_slots", ... }
← 服务端推送: { "type": "tool_result", "tool": "get_free_slots", ... }
← 服务端推送: { "type": "ask_user", "question": "...", "options": [...] }
→ 客户端发送: { "answer": "确认" }
← 服务端推送: { "type": "done", "message": "搞定了..." }
```

断线重连：WebSocket 断开后自动重连（指数退避，最大间隔 30 秒）。重连后重新发送 token 认证。

### 6.3 添加任务

右上角 `+` 按钮，点击弹出表单（底部弹出 sheet 或模态框）：

- 标题（必填）
- 日期（日期选择器，默认当前查看的日期）
- 开始时间（时间选择器）
- 结束时间（时间选择器）
- 描述（可选）
- 关联考试（可选，下拉选择已有考试）

提交后 POST 到 `/api/tasks/`，成功后刷新日历视图。

```json
  "start_url": "/chat",
  "display": "standalone",
  "background_color": "#ffffff",

  "icons": [

    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]

```

### 8.2 Service Worker 策略

| 资源类型 | 策略 | 说明 |
|----------|------|------|
| 静态资源（JS/CSS/图片） | cache-first | Vite 打包带 hash，长期缓存 |
| API 请求（`/api/*`） | network-only | 数据必须实时 |
| WebSocket | 不缓存 | 实时通信 |
| 推送事件 | Service Worker 监听 | 弹系统通知 |

### 8.3 推送订阅流程

1. 用户首次打开 App → 请求浏览器通知权限
2. 用户同意 → 前端获取 push subscription → `POST /api/push/subscribe`
3. 用户拒绝 → "我的"页通知设置显示"未开启"，引导去浏览器设置

### 8.4 推送通知点击

Service Worker 收到 push 事件 → 弹出系统通知（标题 + 内容）。用户点击通知 → 打开 App。

不需要离线功能——核心功能（AI 对话、查日程）都依赖网络。Service Worker 只负责静态资源缓存加速 + 推送接收。

## 9. 状态管理（Zustand）

三个独立 store：

| Store | 职责 | 持久化 |
|-------|------|--------|
| authStore | token、用户信息、登录/登出 | localStorage（token） |
| chatStore | 消息列表、WebSocket 连接、发送消息 | 无（会话内存） |
| calendarStore | 当前日期、课程列表、任务列表 | 无（从 API 获取） |

## 10. API 消费总结

前端消费的后端 API：

| 功能 | 端点 | 页面 |
|------|------|------|
| 注册 | POST `/api/auth/register` | 注册页 |
| 登录 | POST `/api/auth/login` | 登录页 |
| 获取用户信息 | GET `/api/auth/me` | 全局 |
| AI 对话 | WebSocket `/ws/chat` | 聊天页 |
| 上传课表 | POST `/api/schedule/upload` | 聊天页、课表管理 |
| 课程 CRUD | `/api/courses/*` | 日历页、课表管理 |
| 任务 CRUD | `/api/tasks/*` | 日历页 |
| 考试列表 | GET `/api/exams/` | 添加任务表单 |
| 提醒列表 | GET `/api/reminders/` | 通知设置 |
| 推送订阅 | POST/DELETE `/api/push/subscribe` | 通知设置 |
| VAPID 公钥 | GET `/api/push/vapid-key` | 推送订阅流程 |
